How I Build

Engineering Philosophy

Reliability before Intelligence

If a model cannot be monitored, versioned, and rolled back safely, it does not belong in production. Observability and operational safety come first — accuracy follows.

Trade-offs are Explicit

Every system balances latency, cost, accuracy, and operational complexity. I document constraints, evaluate alternatives, and design around known boundaries — not assumptions.

Failure is Assumed

Retries, dead-letter queues, fallback logic, health checks, and structured metrics are built-in — not afterthoughts. Systems degrade gracefully or they fail unpredictably.

Event Analytics Platform — Event-Driven Ingestion Pipeline with Failure Isolation

Personal Project · Go + Kafka + PostgreSQL

Production-grade event-driven data ingestion pipeline designed for correctness under failure. Implements at-least-once delivery with idempotent persistence, intelligent error classification, bounded retry with exponential backoff, and dead-letter queue isolation — all with manual Kafka offset control for crash-safe guarantees.

  • At-least-once delivery with manual offset commits — offsets committed only after successful DB write or DLQ routing; uncommitted messages redelivered on consumer restart
  • Idempotent persistence via INSERT ... ON CONFLICT DO NOTHING — duplicate replays are safe by design
  • Intelligent error classification: transient (exponential backoff + jitter), permanent (immediate DLQ routing), unknown (defaults to transient to avoid data loss)
  • Dead Letter Queue with full forensic metadata: original key/value/offset, error classification, retry count, failure timestamp

Content Risk Scoring Service — Stateless Inference Microservice with Graceful Degradation

Personal Project · FastAPI + Kubernetes + Prometheus

Synchronous text risk scoring microservice serving probabilistic risk scores under strict latency constraints. Model training is offline and versioned; inference is stateless and horizontally scalable. Failures are observable — not silent.

  • Offline versioned model training decoupled from inference; gated CI/CD pipeline for model releases — no hot-push to production
